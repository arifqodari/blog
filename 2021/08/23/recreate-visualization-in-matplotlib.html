Recreate Visualization in Matplotlib

Aug 23, 2021 • 6 min read

Last week I watched an interesting video about recreating a D3.js chart using R ggplot. This is the original chart made using D3.js and this is the recreated chart using ggplot.

In this post, basically I'll go through step by step to recreate the same chart, but using Matplotlib.

# some utility functions

class Head(object):
    
    def __init__(self, lines, fd=sys.stdout):
        self.lines = lines
        self.fd = fd
        
    def write(self, msg):
        if self.lines <= 0:
            return
        
        n = msg.count("\n")
        if n < self.lines:
            self.lines -= n
            return self.fd.write(msg)
        
        ix = 0
        while(self.lines > 0):
            iy = msg.find('\n', ix + 1)
            self.lines -= 1
            ix = iy
            
        return self.fd.write(msg[:ix])
    
    
def pprint_head(to_print, length=10):
    pprint(to_print, stream=Head(length))
    
    
def glimpse(df, rows=10, info=False):
    if info:
        display(df.info())
    else:
        display(df.head(rows))
    return df

Convert json into DataFrame

To make our life easier, we will convert the json data into DataFrame. Once it is a DataFrame, it will be easier to modify and transform the data.

df = (
    # convert json into tabular
    pd.json_normalize(data_json["networkLines"], "line", ["network", "total", "max"])
    .pipe(glimpse, info=True)
    # specify data type for numeric variables
    .astype({"year": "int", "value": "int", "total": "int", "max": "int"})

Pivot the Data into Wide Format

The dataframe above is in long format. It can be noticed that each row represents one observation and each column only represent 1 variable / metric.

However, for visualization purpose, we will pivot the data into a wide format. For each row in the DataFrame, it will represent observations in a single year.

yearly_df = (
    df
    .copy()
    .pivot(columns="network", values="value", index="year")
)

Missing Parts

The result above is already pretty much similar with the original visualization. However, we still have some missing / uncovered parts:

  1. Font family

    The font family I used are not the same as in the original

  2. Rich title

    I found it is not straight forward to write a rich texh format for chart title

  3. Avoid text overlap

    As you might see on the legend texts (right-hand side) that the text are overlapped each other and unreadable.
